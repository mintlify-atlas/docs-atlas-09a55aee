Single Buy is Bancard's primary payment method for one-time transactions. It allows customers to make immediate payments without requiring card tokenization or storage.

## What is Single Buy?

Single Buy is a hosted payment flow where:

- Customers are redirected to Bancard's secure payment interface
- Card details are entered directly on Bancard's servers
- No sensitive card data touches your server
- Payment confirmation is handled via webhooks
- Only PYG (Paraguayan Guaraní) currency is supported

<Info>
Single Buy is ideal for e-commerce checkout flows where customers make one-time purchases without needing to save their payment information.
</Info>

## Generating the Payload

The `single_buy()` method constructs the API request payload:

```python
def single_buy(self):
    formatted_amount = f"{self.monto:.2f}"
    payload = {
        "public_key": self.clave_publica,
        "operation": {
            "token": self.generate_md5_single_buy(
                self.clave_private,
                int(self.shop_process_id),
                f"{formatted_amount}",
                "PYG",
            ),
            "shop_process_id": int(self.shop_process_id),
            "amount": f"{formatted_amount}",
            "currency": "PYG",
            "additional_data": "",
            "description": self.descripcion,
            "return_url": urls.url_join(self.base_url, "/payment/bancard/webhook"),
            "cancel_url": urls.url_join(self.base_url, "/payment/bancard/return"),
        },
    }
    return payload
```

## Token Generation

The security token is generated using MD5 hashing to ensure request authenticity:

```python
def generate_md5_single_buy(self, private_key, shop_process_id, amount, currency):
    # Concatenar los valores
    concatenated = f"{private_key}{shop_process_id}{amount}{currency}"
    
    # Generar el hash MD5
    token = hashlib.md5(concatenated.encode()).hexdigest()
    _logger.info(f"GENERO EL TOKEN SINGLE BUY: {token}")
    return token
```

**Location:** `models/method_single_buy.py:48-55`

<Steps>

<Step title="Generate MD5 Hash">
Compute the MD5 hash of the concatenated string:

```python

```
</Step>

<Step title="Include in Payload">
Add the generated token to the operation object in your API request.
</Step>

<Warning>
**Important:** The amount must be formatted with exactly 2 decimal places (e.g., `150000.00`), and the currency must always be `PYG`. Incorrect formatting will cause token validation to fail.
</Warning>

## Making the API Request

The payment provider sends the payload to Bancard's Single Buy endpoint:

```python
def single_buy(self, monto, descripcion):
    self.ensure_one()
    
    if not all([self.clave_publica, self.clave_private]):
        raise ValidationError(
            _("El proveedor de pago Bancard no está configurado correctamente.")
        )
    
    shop_process_id = self.env["ir.sequence"].next_by_code("bancard.shop.process")
    
    method_single_buy = MethodSingleBuy(
        monto,
        descripcion,
        self.clave_publica,
        self.clave_private,
        self.get_base_url(),
        shop_process_id,
    )
    payload = method_single_buy.single_buy()
    
    response = self._bancard_make_request(payload, "/vpos/api/0.3/single_buy")
    return response.get("process_id"), shop_process_id
```

## Integration with Odoo

The Single Buy method is integrated into Odoo's checkout flow:

```python
class WebsiteSaleSingleBuyCustom(WebsiteSale):
    def _get_shop_payment_values(self, order, **kwargs):
        order = request.website.sale_get_order()
        
        if order:
            single = (
                request.env["payment.provider"]
                .sudo()
                .search([("code", "=", "bancard")], limit=1)
            )
            
            process_id, shop_process_id = single.single_buy(
                order.amount_total, order.name
            )
            
            order.write({
                "process_id": process_id,
                "shop_process_id": shop_process_id,
            })
            request.env.cr.commit()
        
        response = super()._get_shop_payment_values(order, **kwargs)
        return response
```

Bancard Single Buy only supports PYG (Paraguayan Guaraní):

```python
def _get_supported_currencies(self):
    """Override: Bancard solo soporta PYG."""
    supported_currencies = super()._get_supported_currencies()
    if self.code in ["bancard", "catastro"]:
        supported_currencies = supported_currencies.filtered(
            lambda c: c.name == "PYG"
        )
    return supported_currencies
```

## Security Considerations

<Warning>
**Never expose your private key:** The private key should only be used server-side for token generation and must never be sent to the client or included in frontend code.
</Warning>

1. **Token Validation:** Bancard validates the token to ensure request authenticity
2. **HTTPS Required:** All communication must use HTTPS in production
3. **Webhook Security:** Validate `shop_process_id` to ensure it belongs to your system
4. **CSRF Protection:** Webhooks have CSRF disabled (`csrf=False`) as they come from external servers

## Best Practices

<Tip>
Always format amounts with 2 decimal places using `f"{amount:.2f}"` to ensure consistent token generation.
</Tip>

1. **Unique Process IDs:** Use Odoo sequences to generate unique `shop_process_id` values
2. **Idempotency:** Check order state before confirming to prevent duplicate processing
3. **Logging:** Log all API requests and responses for debugging
4. **Error Messages:** Provide clear, user-friendly error messages
5. **Database Commits:** Commit changes before responding to webhooks
