## Overview

The `PaymentProvider` class extends Odoo's payment provider model to integrate Bancard payment gateway functionality. It handles payment operations, card management, and communication with the Bancard API.

**Source**: `models/payment_provider.py:15`

## Class Fields

<ParamField path="code" type="Selection">
  Payment provider code. Adds "bancard" and "catastro" options to the selection.
  
  **Values**: `"bancard"`, `"catastro"`
</ParamField>

<ParamField path="clave_publica" type="Char" required>
  Public key for Bancard API authentication. Required when provider is "bancard".
  
  **Access**: `base.group_system` only
</ParamField>

<ParamField path="clave_private" type="Char" required>
  Private key for Bancard API authentication. Required when provider is "bancard".
  
  **Access**: `base.group_system` only
</ParamField>

<ParamField path="environment" type="Selection">
  Bancard API environment URL.
  
  **Options**:
  - `"https://vpos.infonet.com.py"` - Production
  - `"https://vpos.infonet.com.py:8888"` - Staging (default)
</ParamField>

### single_buy()

Initiates a single payment transaction with Bancard.

**Source**: `models/payment_provider.py:72`

<ParamField path="monto" type="float" required>
  Transaction amount
</ParamField>

<ParamField path="descripcion" type="str" required>
  Payment description
</ParamField>

<ResponseField name="process_id" type="str">
  Bancard's process ID for the transaction
</ResponseField>

<ResponseField name="shop_process_id" type="int">
  Internal shop process ID generated by Odoo sequence
</ResponseField>

```python
# Generate a unique shop process ID
shop_process_id = self.env["ir.sequence"].next_by_code("bancard.shop.process")

# Create the single buy method instance
method_single_buy = MethodSingleBuy(
    monto,
    descripcion,
    self.clave_publica,
    self.clave_private,
    self.get_base_url(),
    shop_process_id,
)

# Generate payload and make request
payload = method_single_buy.single_buy()
response = self._bancard_make_request(payload, "/vpos/api/0.3/single_buy")
```

**Raises**: `ValidationError` if public or private keys are not configured

**Returns**: Tuple of `(process_id, shop_process_id)`

## Usage Example

```python
# Get the Bancard payment provider
provider = self.env['payment.provider'].search(
    [('code', '=', 'bancard')], 
    limit=1
)

# Initiate a payment
process_id, shop_process_id = provider.single_buy(
    monto=150000.00,
    descripcion="Payment for Order #12345"
)

# Register a new card for a user
card_process_id = provider.new_cards(user_id="user_123")

# Retrieve user's cards
cards = provider.get_cards(user_id="user_123")
```

## Configuration Requirements

Before using the PaymentProvider class:

1. Configure `clave_publica` (public key)
2. Configure `clave_private` (private key)
3. Select appropriate `environment` (staging or production)
4. Ensure IR sequences are configured:
   - `bancard.shop.process` for transaction IDs
   - `bancard.card.id` for card registration IDs

## Error Handling

All payment methods validate the presence of API keys before processing:

```python
if not all([self.clave_publica, self.clave_private]):
    raise ValidationError(
        _("El proveedor de pago Bancard no está configurado correctamente.")
    )
```
