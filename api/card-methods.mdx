## Overview

The card management API provides two classes for handling stored payment cards in Bancard:

- **NewCards**: Register new cards for users
- **GetCards**: Retrieve existing cards for users

Both classes handle secure token generation and API payload construction.

### generate_md5_catastro()

Generates the MD5 security token for card operations.

**Source**: `models/method_new_cards.py:38`

<ParamField path="private_key" type="str" required>
  Bancard private API key
</ParamField>

<ParamField path="card_id" type="str" required>
  Unique card identifier
</ParamField>

<ParamField path="user_id" type="str" required>
  Unique user identifier
</ParamField>

<ParamField path="operation" type="str" required>
  Operation type (e.g., "request_new_card")
</ParamField>

<ResponseField name="token" type="str">
  32-character MD5 hexadecimal hash
</ResponseField>

```python
token = method.generate_md5_catastro(
    private_key="priv_xyz789ghi012",
    card_id="card_12345",
    user_id="user_67890",
    operation="request_new_card"
)
```

**Token Generation Algorithm**:

1. Concatenate: `private_key + card_id + user_id + operation`
2. Encode to bytes: `concatenated.encode()`
3. Generate MD5 hash: `hashlib.md5(bytes).hexdigest()`

**Implementation**:

```python
concatenated = f"{private_key}{card_id}{user_id}{operation}"
token = hashlib.md5(concatenated.encode()).hexdigest()
```

**Returns**: MD5 hash string

### API Endpoint

The generated payload is sent to:

```
POST {environment}/vpos/api/0.3/cards/new
```

### Complete Usage Example

```python
from models.method_new_cards import NewCards
import requests

# Initialize the method
method = NewCards(
```

## Security Considerations

### Token Generation

All card operations require secure MD5 token generation:

- **NewCards**: Uses `private_key + card_id + user_id + operation`
- **GetCards**: Uses `private_key + user_id + operation`

### Private Key Handling

- Private keys are never sent in API requests
- Private keys are only used for local token generation
- Tokens must be regenerated for each request

### User Identification

- User IDs must be unique and persistent
- Card IDs must be unique across the system
- Both should be generated using secure sequences

## Integration Example

Complete workflow for card management:

```python
from models.method_new_cards import NewCards
from models.method_get_card import GetCards
import requests

base_url = "https://vpos.infonet.com.py"
public_key = "pub_abc123"
private_key = "priv_xyz789"
user_id = "user_12345"

# Step 1: Register a new card
new_card = NewCards(
    public_key=public_key,
    private_key=private_key,
    base_url="https://mystore.com",
    card_id="card_001",
    user_id=user_id,
)

payload = new_card.new_cards()
response = requests.post(
    f"{base_url}/vpos/api/0.3/cards/new",
    json=payload
)
print(f"Card registration initiated: {response.json()}")

# Step 2: Retrieve user's cards
get_cards = GetCards(
    public_key=public_key,
    private_key=private_key,
    user_id=user_id,
)

payload = get_cards.get_cards()
response = requests.post(
    f"{base_url}/vpos/api/0.3/users/{user_id}/cards",
    json=payload
)

cards = response.json().get("cards", [])
print(f"User has {len(cards)} stored cards")
for card in cards:
    print(f"  - {card['card_masked_number']} ({card['card_brand']})")
```
